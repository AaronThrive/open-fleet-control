## 5. Findings — blockers vs. acceptable deferrals

### Merge blockers
**None.** The feature ships dark (`enabled:false`), the suite is green, security passed, and the regression contract holds.

### Warnings (fix-before-live-flip, NOT merge blockers)

- **W-1 — AC-5 readiness probe has no live HTTP implementation wired.** `src/agent-spawn.js:497` (`reconcile` defaults `probeFn` to `async () => true`) and `:747` (`acquireWorker` same default). No caller in `src/index.js` supplies a real `/health` probe. At the live `enabled:true` flip, readiness would pass vacuously — any started container is treated "ready" without an actual probe. The state machine, N-consecutive-OK logic, and fail-closed drain are all correct and unit-tested via injection; only the live probe wiring is absent. **Action:** wire a real HTTP `/health` (see W-3 for the correct path) probe into `agentSpawn.start()`/`acquireWorker` before the flip.

- **W-2 — AC-15 live free-RAM read (`/proc/meminfo` MemAvailable) not wired.** `src/agent-spawn.js:227` only consults `readMemAvailableFn` when one is passed; `src/index.js` never passes one. The count-ceiling and projected-footprint arms of the governor are live, but the live MemAvailable arm is dormant in production. Logic + tests are correct via injection. **Action:** wire a `/proc/meminfo` reader into the controller before the flip so the governor refuses on real low-RAM, not just on the projected count.

- **W-3 — health-path seam mismatch (cross-repo).** The worker runtime serves health at `/api/health` (OFC dashboard `src/index.js:645`; `src/agent-locator.js:35` declares `OFC_DASHBOARD_HEALTH_PATH = "/api/health"`; stack `compose.worker-pool.yaml.template:85` healthcheck uses `/api/health`). But the spawn controller registers each worker with `healthPath: "/health"` because `src/agent-spawn.js:706` defaults to `/health` and the compose template sets **no** `com.ofc.pool.healthPath` label override. At live verification the mesh poller would GET `/health`, receive 404, mark the worker `unreachable`, and the registration would not sustain. **Action (pick one) before the flip:** (a) add `com.ofc.pool.healthPath: "/api/health"` to the worker labels in the stack compose template, or (b) change the OFC `workerHealthPath` default to `/api/health`. Unit tests inject `probeFn` so they cannot catch this — it is a live-only failure.

### Suggestions (nice-to-have)

- **S-1 — S-AC-9 smoke test is not invoked by install or CI.** `docker/smoke-test-node-sqlite.js` exercises the exact AC-9 verbs but `scripts/install.sh` never runs it (`grep -c smoke install.sh` = 0). The AC accepts a manual run, but invoking it once inside the freshly-built worker image during install (or a CI step) would make the Node-pin guarantee self-enforcing rather than a manual checklist item.

- **S-2 — Q1 (shared-vs-dedicated worker image) resolved to "reuse OFC image in advisor mode."** Documented in `docker/Dockerfile:5-31`. Acceptable and within the PRD's deferred-to-build latitude; noted only so the lean-default decision is on record for the maintainer.
